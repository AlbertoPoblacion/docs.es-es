# <a name="xml-literal-cannot-appear-here-unless-it-is-enclosed-in-parentheses"></a>Aquí no puede aparecer un literal XML a menos que se incluya entre paréntesis
Una declaración literal de XML se utiliza en una expresión en una ubicación que es ambigua para el compilador de Visual Basic. Es decir, no se puede determinar el compilador de Visual Basic si el menor-que (<) del carácter está pensado como operador comparativo o para iniciar un literal XML. El siguiente fragmento de código muestra un ejemplo:  

```vb  
' Generates an error.  
Dim queryResult = From element In elements _  
                  Where <sample>Value</sample> = "Value" _  
                  Select element  
```  
  
 **Identificador de error:** BC31198  
  
## <a name="to-correct-this-error"></a>Para corregir este error  
  
-   Incluya la declaración de literal XML entre paréntesis, tal como se muestra en el ejemplo siguiente:  
  
```vb  
Dim queryResult = From element In elements _  
                  Where (<sample> Value</sample>) = "Value" _  
                  Select element  
```  
  
-   Modifique el código para hacer referencia a un literal XML existente, tal como se muestra en el ejemplo siguiente:  
  
```vb  
Dim queryResult = From element In elements _  
                  Where e.<sample>.Value = "Value" _  
                  Select element  
```
